# 部署文档

## 1. 文档目标

本文档说明如何从零开始在本地机器上运行当前版本的 DeepLung 工程，包括：

- 环境准备
- Python 依赖安装
- 前端依赖安装
- `mock` 模式启动
- `MONAI + GPU` 模式启动
- 冒烟测试与常见问题排查

当前项目推荐的启动方式是本地开发模式，而不是先用 Docker 全量部署。原因是：

- 课程项目更容易调试
- GPU 推理在本地 conda 环境中更直接
- 当前仓库的业务数据仍以原型方式组织，先跑通服务更重要

## 2. 运行前提

建议准备以下环境：

- Windows 10/11
- Python 3.11
- Conda
- Node.js 18+
- Git
- NVIDIA GPU 和可用 CUDA 版 PyTorch

如果要启用 MONAI GPU 推理，至少需要：

- 已安装 CUDA 版 `torch`
- `python -c "import torch; print(torch.cuda.is_available())"` 返回 `True`

## 3. 推荐环境准备流程

### 3.1 创建虚拟环境

```powershell
conda create -n deeplung -y python=3.11
conda activate deeplung
```

### 3.2 先确认 GPU 版 torch

如果你的环境里已经装好 GPU 版 `torch`，先确认：

```powershell
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.cuda.device_count())"
```

你当前环境的目标结果应该类似：

```text
2.10.0+cu128
True
1
```

### 3.3 如需补装 torchvision

如果后续 MONAI 推理时报缺少 `torchvision`，请在同一个 `deeplung` 环境中安装与现有 `torch` 匹配的 `torchvision`。建议优先使用 PyTorch 官方提供的安装方式，避免破坏现有 GPU 版 `torch`。

```powershell
$env:MONAI_INFER_CONFIG_RELPATH='configs/inference.json'

$env:FALLBACK_TO_MOCK_ON_ERROR='true'

```

## 9. 常见问题

### 9.1 `No module named xxx`

说明当前虚拟环境依赖不完整。先运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/setup_deeplung_env.ps1 -InstallMonai
```

### 9.2 `torch.cuda.is_available()` 为 `False`

说明虽然机器有 NVIDIA GPU，但当前 Python 环境里的 `torch` 不是可用的 CUDA 版本，或驱动 / CUDA 运行环境不匹配。

### 9.3 MONAI 在 CPU 下很慢

这是正常现象。3D CT 检测在 CPU 下适合开发验证，不适合正式演示。答辩时建议优先使用 GPU 模式。

### 9.4 前端无法启动

先确认：

```powershell
npm --version
```

如果报错，说明 Node.js 未安装或终端未刷新 PATH。

### 9.5 AI 服务启动但推理失败

优先查看：

- 根目录下的 `ai-engine.runtime.err`
- 根目录下的 `ai-monai.runtime.err`

## 10. 当前部署边界

**可运行的课程项目原型，而不是临床生产系统。**
