# 肺结节检测平替方案调研

## 目标

在现有 DeepLung 架构下，替换不可用模型，优先满足：
1. 可快速落地
2. 有公开实现
3. 兼容 3D CT 检测任务

## 方案 A（推荐）：MONAI Lung Nodule Bundle

- 来源：MONAI 官方 Model Zoo 对应 bundle
- 特点：提供可下载模型 bundle，推理流程有官方说明，工程化友好
- 适用：你要尽快跑出真实检测结果并接入现有 FastAPI

参考：
- https://huggingface.co/MONAI/lung_nodule_ct_detection
- https://docs.monai.io/en/latest/integration_pipeline.html

## 方案 B：nnDetection（强基线）

- 来源：MIC-DKFZ 官方开源 3D 医学目标检测框架
- 特点：社区认可度高，适合做高质量训练与实验
- 代价：部署门槛更高，通常需要重新训练/微调

参考：
- https://github.com/MIC-DKFZ/nnDetection

## 方案 C：自训路线（LUNA16 / LIDC-IDRI）

- 特点：数据可公开获取，利于课程/科研复现
- 代价：训练和验证周期更长

参考：
- https://luna16.grand-challenge.org/

## 建议结论

如果你的目标是“先能跑，再优化”：
1. 先走方案 A（MONAI bundle）完成端到端联调。
2. 后续若要追求效果上限，再切换方案 B（nnDetection）进行重训。
